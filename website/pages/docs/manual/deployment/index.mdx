
Deploying Hasura GraphQL engine
===============================

Table of contents

Note

This section talks in depth about deploying the Hasura GraphQL engine
for a **production-like environment**. If you would simply like to take
the Hasura GraphQL engine for a quick spin, choose from our
`Getting started guides <getting_started>`{.interpreted-text
role="ref"}.

Deployment guides
-----------------

The Hasura GraphQL engine is a binary that is shipped as a Docker
container.

Choose from the following guides to deploy the Hasura GraphQL engine and
connect it to a Postgres database:

-   `Deploy using Heroku <deploy_heroku>`{.interpreted-text role="ref"}
-   `Deploy using Docker <deployment_docker>`{.interpreted-text
    role="ref"}
-   `Deploy using Kubernetes <deploy_kubernetes>`{.interpreted-text
    role="ref"}

You can also check `guides_deployment`{.interpreted-text role="ref"} for
more specific examples.

Custom Docker images or binaries

If you need a custom Docker image or binary for GraphQL engine, please
see `this page <custom_docker_image>`{.interpreted-text role="ref"}

Configuration
-------------

By default, Hasura GraphQL engine runs in a very permissive mode for
easier development. Check out the below pages to configure the Hasura
GraphQL engine for your production environment:

-   `securing_graphql_endpoint`{.interpreted-text role="ref"}
-   `postgres_permissions`{.interpreted-text role="ref"}
-   `GraphQL engine server configuration <hge_flags>`{.interpreted-text
    role="ref"}
