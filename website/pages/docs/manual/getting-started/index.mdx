
Getting started
===============

Table of contents

To use the Hasura GraphQL engine, you need to:

1.  Run the Hasura GraphQL engine with access to a Postgres database
2.  Use the Hasura console (an admin UI) that connects to the Hasura
    GraphQL engine to help you build your schema and run GraphQL queries

/img/graphql/manual/getting-started/running-hasura.png

Get started from scratch
------------------------

-   `Using Heroku <heroku_simple>`{.interpreted-text role="ref"}
    **(recommended)**: Get started in under 60 seconds with no setup
    required to host the Hasura GraphQL engine and Postgres on Heroku\'s
    free tier.
-   `Using Docker <docker_simple>`{.interpreted-text role="ref"}: Run a
    **local development** setup that sets up both the Hasura GraphQL
    engine and Postgres using Docker.

Get started using an existing database
--------------------------------------

-   `Using Heroku <heroku_existing_db>`{.interpreted-text role="ref"}:
    Run on Heroku using an existing Heroku Postgres database.
-   `Using Docker <deployment_docker>`{.interpreted-text role="ref"}:
    Run as a docker container and connect to an existing Postgres
    database.
-   `Using Kubernetes <deploy_kubernetes>`{.interpreted-text
    role="ref"}: Run on Kubernetes and connect to an existing Postgres
    database.

Supported Postgres versions

Hasura GraphQL engine supports **Postgres 9.5 and above**
